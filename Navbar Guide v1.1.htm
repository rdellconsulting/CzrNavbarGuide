Just about every website you see these days will almost certainly contain a Navbar to enable the User to navigate around the various Pages you have built. By now, you will be familiar with Customizr's own Navbar, part of its' unique style.

It is a common request on the Forum to customize the Navbar, and you will find a number of Snippets that address this subject.

This Guide is intended to take these Snippets a bit further and show you ideas on how you might change the look-and-feel of your Navbar in order to personalize the style of your website.

Basics required

Before we start, let's make sure that you understand the basics. If you are an Advanced User then feel free to skip this next bit.

  • HTML: Have to assume you know this or you wouldn't be reading this Guide.
  • CSS: Most of this Guide will cover this area. You may find these Navbar Snippets useful as an Introduction.
  • PHP: You don't need to understand PHP, but some of the Snippets use PHP so make sure you cut&paste accurately.
  • Javascript: May need to add some Javascript for some effects. Again, no need to understand, just cut&paste accurately.
  • Child Theme: You must know by now that we highly recommend using a child theme. Do not hack core files. For more information, read this Guide.

The Navbar Box

Positioning the Navbar Box

There are 2 simple ways in CSS to reposition the Navbar:

margin|margin-top|margin-right|margin-bottom|margin-left

/* Select one of the margin statements below */
.navbar.notresp navbar-inner {
margin:             0px 0px;                /* Default position */
margin-top:         20px;                   /* Move down by 20px */
margin-bottom:      -20px;
margin-bottom:      20px;                   /* Move up by 20px */
margin-top:         -20px;
margin-left:        20px;                   /* Move right by 20px */
margin-right:       -20px;
margin-right:       20px;                   /* Move left by 20px */
margin-left:        -20px;
margin:             20px 0px 0px 20px;      /* Move down 20px, right 20px */
}

Tip: the margin selector typically uses 1, 2 or 4 dimensions as follows:

margin:             10px;                   /* Adds 10px to top, right, bottom, left */
margin:             0px 10px;               /* Adds 0px to top, bottom and 10px to right, left */
margin:             0px 10px 20px 30px;     /* Adds 0px to top, 10px to right, 20px to bottom, 30px to left */

position:relative|absolute

.navbar.notresp navbar-inner {
position:           relative;
top:                0px;                    /* Default position */
left:               0px;                    /* Default position */
}

So with similar logic to the margin selector, the Navbar can be moved around the page by adjusting the top/bottom and left/right dimensions:

top:                20px; left: 30px;       /* Down 20px, Right 30px */
bottom:             -20px; right: -30px;
top:                -60px; left: -40px;     /* Up 60px, Left 40px */
bottom:             60px; right: 40px;

But this code can be improved to use with Responsiveness, ie working on different viewports. As Customizr is built around Twitter Bootstrap 2.3.2, most of the CSS is built in, and we need to customize the defaults.

For height we use px, but for width we use %.

So the code above becomes:

.navbar.notresp navbar-inner {
position:           relative;
top:                0px;                    /* Default position */
left:               0%;                     /* Default position */
}

And the Navbar can be moved around the page by adjusting the top/bottom and left/right dimensions with positive/negative values:

top:                20px; left: 30%;        /* Down 20px, Right 30% */
bottom:             -20px; right: -30%;
top:                -60px; left: -40%;      /* Up 60px, Left 40% */
bottom:             60px; right: 40%;

/*  Adjust Navbar Box settings      */

min-height:         40px;                   /* Change [40]px */

padding-right:      20px;                   /* Change [20]px */

Hiding the Navbar Box

You may wish to hide the Navbar Box surrounding the menu items:

.navbar.notresp .navbar-inner {
display:            none;                   /* Hide the Navbar Box */
}

Adjusting the white space between the colored & grey bars

You may wish to adjust the whitespace between the two bars in the Header:

/* Select one height setting */
.tc-header {
min-height:         45px;                   /* Adjust [45]px to suit */
height:             60px;                   /* Adjust [60]px to suit */
max-height:         75px;                   /* Adjust [75]px to suit */
}

Removing the top/bottom bars

You may wish to remove the colored & grey bars at the top/bottom:

.tc-header {
border-top:         none;                   /* Remove colored bar */
border-bottom:      none;                   /* Remove grey bar */
}

Menu Items

These Menu items are setup using Dashboard>Appearance>Menus

We'll start by looking at Level 1 items, ie without any sub-menus.

Shading Menu Items

For this we need to use the following selector/properties:

/* Navbar 1 */
.navbar.notresp .nav > li > a {
border:             1px solid green;        /* Add a border */
border-radius:      4px 4px 4px 4px;        /* Rounded corners */
background-color:   yellow;                 /* Change yellow */
background-image:   linear-gradient(to bottom, white, yellow);    /* Change white, yellow */
background-image:   -webkit-linear-gradient (top, white, yellow); /* Change white, yellow */
background-repeat:  repeat-x;               /* Add shading */
}

And change color when mouse hovers:

.navbar.notresp .nav > li > a:hover {
border:             1px solid green;        /* Add a border */
border-radius:      4px 4px 4px 4px;        /* Rounded corners */
color:              yellow;                 /* Yellow Text on hover */
background-color:   darkgreen;              /* Change darkgreen */
background-image:   linear-gradient(to bottom, darkgreen, #9DB67E);    /* Change darkgreen, #9DB67E Light Green */
background-image:   -webkit-linear-gradient (top, darkgreen, #9DB67E); /* Change darkgreen, #9DB67E Light Green */
background-repeat:  repeat-x;               /* Add shading */
}

Tip: Because CSS executes serially, the hover code above need only contain the delta changes:

.navbar.notresp .nav > li > a:hover {
color:              yellow;                 /* Yellow Text on hover */
background-color:   darkgreen;              /* Change darkgreen */
background-image:   linear-gradient(to bottom, yellow, blue);    /* Change yellow, blue */
background-image:   -webkit-linear-gradient (top, yellow, blue); /* Change yellow, blue */
}

Changing the gap between Menu Items

For this we need to use the margin property:

    We now have two embedded unordered lists added, and need to address each property.

    Given that Level 1 items were addressed with: 
    .navbar.notresp .nav > li > a {}
.navbar.notresp .nav > li > a:hover {}
.navbar.notresp .nav > li.current-menu-item > a,.dropdown-menu > li.current-menu-ancestor > a {}

    .dropdown-menu > li > a {}

.dropdown-submenu > li > a {}
.dropdown-submenu > li > a:hover {}
.dropdown-submenu > li.current-menu-item > a,.dropdown-menu > li.current-menu-ancestor > a {}

    Responsiveness

    If you are unfamiliar with the concept of Responsiveness, then start by reviewing this Snippet

    Twitter Bootstrap defines the point at which Customizer changes the menu structure. At 979/980px width, the menu reduces to a 3-bar button. By clicking on the button, Level 2+ menu items are revealed.

    Just like the full navbar, these menu items need to be styled using similar CSS properties. A new set of selectors is now required:

    .navbar.notresp .nav {}                     /* Non-responsive */
.navbar.resp    .nav {}                     /* Responsive     */
.navbar         .nav {}                     /* Either non-resp or resp */

    At this point, the term 'specificity' becomes very important. For a simple explanation, try this article.

    When adjusting Navbar styling, you will need to decide whether the styling applies to both non-responsive and/or responsive items. For example, consider the impact of these 3 selectors:

    .navbar .nav > li > a {
color:              blue;                   /* Generic menu item */
}
.navbar.notresp .nav > li > a {
color:              red;                    /* Fullsize menu item */
}
.navbar.resp .nav > li > a {
color:              green;                  /* 3-bar menu item */
}

    Take a look at the CSS Structure for the Navbar in the Appendix and it should become clearer to you.
